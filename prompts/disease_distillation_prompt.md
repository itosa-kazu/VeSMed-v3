# VeSMed 疾患データ蒸留プロンプト

## 使い方
1. 本プロンプトをAIに投入
2. `{VARIABLE_LIST}` の部分に `variable_list.txt` の内容を貼り付け
3. `{DISEASE_NAME_JA}` に疾患名（日本語）を指定
4. `{DISEASE_ID}` に疾患ID（例: D999）を指定

---

## プロンプト本文

```
あなたは臨床医学のエキスパートです。以下の診断支援システム(VeSMed)に新しい疾患を追加するためのデータを生成してください。

# 対象疾患
- 疾患ID: {DISEASE_ID}
- 疾患名: {DISEASE_NAME_JA}

# 利用可能な変量一覧
以下のフォーマットで変量が定義されています: ID:name[state1|state2|...]

{VARIABLE_LIST}

# 出力フォーマット

以下の3つのJSONブロックを **一度に** 出力してください。

## 1. step2_edges（辺の定義）

この疾患が関連する全ての辺をリストで出力。
辺の種類: D→S(症状), D→E(身体所見), D→L(検査), D→T(時間), R→D(リスク因子→疾患)

```json
{
  "disease_id": "{DISEASE_ID}",
  "disease_name": "english_snake_case",
  "disease_name_ja": "{DISEASE_NAME_JA}",
  "edges": [
    {
      "from": "{DISEASE_ID}",
      "to": "変量ID",
      "from_name": "english_snake_case",
      "to_name": "変量のname",
      "reason": "臨床的根拠（日本語、1文）"
    },
    {
      "from": "R変量ID",
      "to": "{DISEASE_ID}",
      "from_name": "R変量のname",
      "to_name": "english_snake_case",
      "reason": "リスク因子の根拠（日本語、1文）"
    }
  ]
}
```

**辺の選定基準:**
- その疾患の患者に **臨床的に有意な頻度で** 出現する所見のみ（頻度<5%の偶発所見は除外）
- 鑑別に **役立つ** 所見を優先（特異度が高い所見 > 感度が高いだけの所見）
- R→D辺: その疾患に疫学的に有意なリスク因子のみ
- 一般的すぎる所見（例: ほぼ全疾患で発熱する場合のE01）も含めるが、CPTで差別化

## 2. noisy_or_params（条件付き確率）

各辺に対応するCPTを、変量ごとにまとめて出力。

```json
{
  "noisy_or_params": {
    "変量ID": {
      "{DISEASE_ID}": {
        "state1": 確率,
        "state2": 確率
      }
    }
  }
}
```

**CPT値の鉄則:**
- 各変量の全state確率の合計 = 1.0（厳密に）
- 値は **実際の臨床データ(n/N)** に基づく。"common"→0.7のような主観変換は禁止
- 情報源: UpToDate > Systematic Review > 教科書 > Case Series の優先順で参照
- 確率0%は使わない（最低0.001）
- 確率100%は使わない（最高0.999）
- absent/presentの2値変量: absentの値 = 1 - presentの値
- 多値変量（例: E01体温）: 各stateにこの疾患の典型的な分布を設定

**CPT値の具体例（参考）:**
- 「ほぼ必発」= 0.85-0.95
- 「多い」= 0.6-0.8  
- 「半数程度」= 0.4-0.6
- 「時々」= 0.15-0.35
- 「まれ」= 0.03-0.1
- 「ほぼない」= 0.001-0.02

## 3. disease_prior（疾患の事前確率）

R01(年齢)とR02(性別)の事前確率を出力。

```json
{
  "disease_prior": {
    "{DISEASE_ID}": {
      "R01": {
        "0_1": 確率,
        "1_5": 確率,
        "6_12": 確率,
        "13_17": 確率,
        "18_39": 確率,
        "40_64": 確率,
        "65_plus": 確率
      },
      "R02": {
        "male": 確率,
        "female": 確率
      }
    }
  }
}
```

R01/R02の値は **発熱外来を受診した患者のうち、その疾患である確率** を年齢・性別ごとに設定。
全年齢・全性別の加重平均が、その疾患の一般的な有病率に近似するように。

# 出力ルール

1. 上記3つのJSONを順に出力
2. 各CPT値の横に、根拠となる情報源を簡潔にコメント
3. 不確実な値には ⚠️ を付与し、根拠の弱さを明示
4. 既存変量に存在しない所見が重要な場合、「新変量候補」として別途リスト化
5. 確定診断検査（例: 生検、培養の最終結果）は辺に含めない
```

---

## 使用例

```
対象疾患: D999 溶血性尿毒症症候群(HUS)
```

→ AIが E01(体温), L01(WBC), L02(CRP), S13(悪心嘔吐), S12(腹痛), S14(下痢) 等への辺 + CPT + R01/R02 priorを一括出力

## 注意事項
- AIの出力は必ず人間がレビューする
- CPT値は文献で検証する（6大鉄則: CPT変更は文献必須）
- 新変量候補が出た場合は add-variables ワークフローで追加
- 出力後は `python3 validate_edges.py` で整合性チェック
