# VeSMed V3 模型问题分析 — 为什么加边反而变差？

## 给另一个AI的背景介绍

我们在做一个**发热鉴别诊断**的贝叶斯网络（BN）系统：
- 104种发热相关疾病（D01~D104）
- 275个变量（症状S、体征E、检验L、风险因子R、时间T等）
- 1121条因果边（疾病→症状/检验等）
- 推理引擎采用 **Noisy-OR + log-Likelihood Ratio** 打分
- 76个真实文献case做测试，当前Top-1命中率42%（31/73 in-scope）

## 碰到的问题

用程序对照教科书，发现有289条"缺失边"（例如：某些疾病应该导致WBC升高但没建边）。全部加上后，精度**反而下降**：

| 指标 | 加边前 (1121边) | 加289边后 (1410边) |
|------|----------------|-------------------|
| Top-1 | 31/73 (42%) | 29/73 (40%) |
| Top-3 | 55/73 (75%) | 50/73 (68%) |
| FATAL(Top-10外) | 4 | 7 |

这289条边医学上是**正确的**——这些疾病确实会导致这些症状。但加上后诊断能力变差了。

## 推理引擎核心公式

```python
# bn_inference.py 第241-303行
def infer(evidence, risk, diseases, disease_children, noisy_or, root_priors):
    for d in diseases:
        prior = get_prior(d, root_priors, risk)
        log_post = math.log(prior)

        for var_id, obs_state in evidence.items():
            p_leak = leak[resolved_state]        # "无病"时的基础概率

            if d in parent_effects:
                p_d = parent_effects[d][resolved_state]  # 疾病d导致该状态的概率
            else:
                p_d = p_leak   # 无边 → LR=1 → 中性（不影响评分）

            log_post += log(p_d) - log(p_leak)   # log-LR累加

        log_posteriors[d] = log_post

    # softmax归一化后排序
    return sorted(posteriors.items(), key=lambda x: -x[1])
```

**关键设计**：
- 疾病d对变量v**无边**时：`p_d = p_leak`，LR=1，log(LR)=0（完全中性）
- 疾病d对变量v**有边**时：LR = p_d / p_leak（可能 >1 boost 或 <1 penalty）
- 最终得分 = log(先验) + 所有证据变量的 log-LR 之和

## 问题的根源分析

### 1. leak值代表的是"健康人"基线，不是"发热病人"基线

这是我们发现的最核心问题。

```
S09 (恶寒战栗): leak = 0.03,  全疾病平均CPT = 0.62  → 比值 20.8倍
S13 (恶心):     leak = 0.05,  全疾病平均CPT = 0.61  → 比值 12.2倍
S05 (头痛):     leak = 0.08,  全疾病平均CPT = 0.62  → 比值  7.8倍
S46 (厌食):     leak = 0.10,  全疾病平均CPT = 0.67  → 比值  6.7倍
S07 (倦怠):     leak = 0.10,  全疾病平均CPT = 0.66  → 比值  6.6倍
```

以S09（恶寒战栗）为例：
- leak = 0.03（健康人出现恶寒的概率 = 3%）
- 但发热病人出现恶寒的概率通常 60-80%
- 所以任何有S09边的疾病都得到 LR = 0.65/0.03 ≈ **21倍boost**
- 当25个疾病都有S09边时，所有疾病都得到同样巨大的boost → **鉴别力为零，但噪声巨大**

问题在于：这个系统专门用于**发热病人的鉴别诊断**。所有104种疾病都是发热疾病。比较的基准应该是"一般发热病人"，而不是"健康人"。

### 2. 非特异性变量的边密度灾难

加289边后，最严重的几个变量：

| 变量 | 原有边数 | 加边后 | 占102疾病比 |
|------|---------|--------|------------|
| T01 (发热持续期间) | 52 | 100 | **98%** |
| L01 (白血球数) | 48 | 84 | 82% |
| L02 (CRP) | 77 | 91 | 89% |
| S46 (厌食) | 11 | 33 | 32% |

当98%的疾病都有T01的边，T01就完全丧失了鉴别能力——但因为leak设得很低，每条边仍然在制造很大的LR波动（噪声）。

### 3. 根因对比

| 比较对象 | 当前模型 (Noisy-OR log-LR) | 临床医生思维 |
|---------|---------------------------|-------------|
| 基准线 | leak = P(症状\|健康人) = 很低 | "发热病人大概率有恶寒，不看这个" |
| 非特异性症状 | LR = P(症状\|疾病)/leak ≫ 1 → 巨大boost | "无鉴别价值，跳过" |
| 证据权重 | 所有边等权相加 | 按特异性动态加权 |
| 症状组合 | 各变量独立(Naive Bayes假设) | 看syndrome/pattern |

## 从真实临床倒推：医生到底怎么鉴别诊断的？

上面的分析是从模型数学出发的，但最终要落地到真实临床。让我们反过来想：一个有经验的感染症医生，面对发热患者时，脑子里的推理过程到底是什么？

### 临床鉴别诊断的真实思维流程

**第1步：先看"大框架"（几秒钟）**
- 急性 vs 亚急性 vs 慢性？→ 决定了候选疾病的大类
- 病人"看起来重不重"？→ 决定了urgency
- 这一步用的就是那些"没有鉴别力"的东西（发热模式、全身状态），但不是用来鉴别，而是用来**分类/分层**

**第2步：抓"关键所见"（核心鉴别步）**
- 不是看所有症状，而是找**1-3个最特异的所见**
  - 双峰热（double quotidian）→ 几乎只有Still和疟疾
  - 传染性单核球症三征（发热+咽頭痛+頸部リンパ節腫大）→ EB病毒
  - Eschar（焦痂）→ つつが虫病/リケッチア
  - 比目魚筋圧痛 → DVT/PE
- 这一步的本质：**不是"这个症状支持什么病"，而是"什么症状只有少数几个病才有"**

**第3步：用"阴性所见"排除（常被忽略的重要步骤）**
- "这个病人如果是IE（心内膜炎），应该有心杂音——但没有"→ IE可能性降低
- "如果是SLE，应该有ANA阳性——但阴性"→ SLE基本排除
- 这一步的关键：**不是"证据支持什么"，而是"应该有但没有"**

**第4步：Risk factor做贝叶斯先验调整**
- 渡航歴、動物接触、年齢、性別、基礎疾患
- 这一步确实是先验概率调整，当前模型已有

### 当前模型 vs 临床思维的差距

| 临床步骤 | 当前模型表现 | 差距 |
|---------|------------|------|
| 第1步：分类/分层 | 没有分层概念，104种病平等比较 | 缺少疾病分类层次 |
| 第2步：抓特异性所见 | 所有边等权累加，特异性所见被非特异性噪声淹没 | **最核心的差距** |
| 第3步：阴性排除 | leak设为健康人基线，"没有心杂音"对IE的排除力不够 | 阴性证据权重不足 |
| 第4步：Risk先验 | 已实现 | 基本OK |

### 这意味着什么？

**核心洞察**：问题不在于"要不要加边"，而在于模型无法表达"这条边的鉴别意义是什么"。

在临床中：
- "发热+头痛" — 几乎所有发热病都有，医生**自动忽略**
- "发热+焦痂" — 极少数病有，医生**立刻锁定**

两者都是"疾病→症状"的因果边，但临床意义完全不同。当前模型把它们等价处理。

所以**真正的问题**不是leak值对不对、要不要加权——而是模型需要能区分：
1. **筛查性所见**（screening）：帮助缩小范围但不能确诊
2. **鉴别性所见**（discriminating）：直接指向少数疾病
3. **排除性所见**（ruling-out）：阴性结果排除特定疾病

## 可能的解决方向

### 方向A：修正leak为"发热人群基线"（不改模型结构）

**思路**：把leak从"健康人基线"改为"发热人群加权平均"。

```
新leak(v) = Σ_d P(v|d) × prior(d)   # 即所有疾病的CPT加权平均
```

**效果**：
- 非特异性症状：新leak ≈ 大多数疾病的CPT → LR ≈ 1 → 自动中性化
- 特异性症状：新leak仍然低 → LR仍然大 → 保持鉴别力

**优点**：数学上有道理（条件在"发热人群"上做鉴别），不改模型架构
**疑问**：改了leak后Noisy-OR的概率语义还对不对？leak在Noisy-OR里有特定含义（"模型外原因导致症状出现的概率"）

### 方向B：双层推理（先筛后鉴别）

**思路**：
1. 第一层：用所有边（含非特异性）筛选candidate diseases（比如Top-20）
2. 第二层：只用高特异性边做精确鉴别排序

**优点**：模拟临床思维的"先广后窄"
**缺点**：两层之间的切分标准不好定

### 方向C：引入边的"鉴别力系数"（改推理引擎）

```python
# 改造后的推理公式
score(d) = log(prior) + Σ_v [ disc(v) × (log P(v|d) - log P(v|leak)) ]

# disc(v) = 变量v的鉴别力，由信息增益或IDF-like指标计算
# 非特异性变量 disc ≈ 0 → 边存在但几乎不影响评分
# 特异性变量 disc ≈ 1 → 正常影响
```

**优点**：保留所有边的医学完整性，同时抑制噪声
**缺点**：disc(v)怎么计算？人工定还是自动算？有点ad-hoc

### 方向D：放弃Naive Bayes独立假设，引入症状间相关性

当前模型假设所有症状在给定疾病下**条件独立**。但临床上：
- "WBC高 + CRP高 + 左方移動" → 作为pattern有意义
- "WBC高" 单独看 → 没意义

可以用：
- **Tree-Augmented Naive Bayes (TAN)**：允许症状间有一条依赖边
- **中间变量（Syndrome节点）**：D→DIC→{血小板低,出血,D-dimer高}
- **完整BN**：全建模（但275变量的junction tree推理可能爆炸）

### 方向E：用学习替代手工参数

当前所有CPT都是LLM基于教科书估计的，没有用真实数据学习。
- 用MIMIC-IV等真实数据学习CPT参数
- 或用EM算法从case数据做参数估计
- 至少可以检验LLM估计的CPT和真实分布差多远

### 方向F：从临床思维倒推的分层模型

**核心思路**：不改Noisy-OR，但把变量分成三类，推理时分步处理：

```
变量分类：
  Type-S (Screening): T01发热持续, L01 WBC, L02 CRP, S09恶寒, S07倦怠...
    → 几乎所有发热病都有，用于粗筛（排除明显不符的大类）

  Type-D (Discriminating): E12皮疹类型, T03发热パターン, E15心雑音, L04胸部X線...
    → 只有少数疾病有特异表现，用于精确鉴别

  Type-R (Ruling-out): L09血培, L18 ANA, L06迅速インフル, E15心雑音(absent)...
    → 阴性结果用于排除特定疾病
```

**推理流程**：
```
Step 1: 先验 × Risk → 104个疾病的初始概率
Step 2: Type-D (鉴别性所见) → 大幅更新后验（LR可以很大）
Step 3: Type-R (排除性所见) → 阴性结果排除特定候选
Step 4: Type-S (筛查性所见) → 微调（LR被压缩到接近1）
```

**和方向A的区别**：方向A是改leak参数，方向F是改推理逻辑。方向F更接近临床思维，因为它不是"所有边等权求和"，而是"先看重要的，再看次要的"。

**实现方式**：可以在step1的变量定义中加一个`diagnostic_role`字段：
```json
{"id": "E12", "name": "skin_findings", "diagnostic_role": "discriminating", ...}
{"id": "S09", "name": "chills", "diagnostic_role": "screening", ...}
```
推理引擎中对不同类型的变量使用不同的LR缩放系数。

## 值得讨论的更深层问题

1. **Noisy-OR的"无边=中性"设计对不对？**
   在临床中，"疾病D不会导致症状V"也是有鉴别意义的信息。但当前模型中，无边 → p_d = p_leak → LR=1 → 完全忽略。如果病人有V而D不应该导致V，这其实应该是对D的**惩罚**。

2. **prior的设定对结果影响有多大？**
   当前所有疾病的prior是1/104（均匀先验）还是根据流行率设定？这会影响泛用疾病（普通感冒）vs 罕见病（Castleman）的竞争。

3. **证据的"量"vs"质"**
   当前模型中，提供更多证据不一定更好（因为非特异性证据是噪声）。但临床上，更多信息通常更好。这说明模型对信息的利用效率有问题。

## 我目前的判断

- **方向A（改leak）**：最快能试，但只是参数修正，没解决结构问题
- **方向F（分层推理）**：最贴近临床思维，实现难度中等，可能是最佳折中
- **方向D（中间变量）**：正确的长期方向，工作量大
- **方向E（数据学习）**：终极方案，受限于数据获取
- **方向A + F组合**：先改leak纠正基线偏差，再引入变量分类做分层推理，可能是最好的渐进路线
