**疾患リストを渡されたら、連続で追加する。1つずつ停止して確認を求めない。**

**Why:** ユーザーは疾患追加の自動化・効率化を望んでいる。毎回停止して次を聞くのは非効率。

**How to apply:**
- リスト内の各疾患を順番にadd-diseases workflowで処理
- 各疾患ごとにcommit（回帰テストで問題がなければ）
- 全部終わってからまとめて報告
- ただしFATALや大幅退化が出たら一旦停止して報告
