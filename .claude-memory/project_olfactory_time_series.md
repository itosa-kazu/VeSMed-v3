## 核心発見: 侧抑制と適応は同一操作

嗅球の顆粒細胞(granule cell)は**2つの役割を同時に**担う:
1. **快照内**: 侧抑制（異なる気味成分間の競争）
2. **快照間**: 適応（前回の嗅探の記憶を減算）

```
僧帽細胞出力 = 新入力 − 顆粒細胞抑制
             = 新入力 − 旧状態の記憶
             = 変化量（差分）
```

**差分計算は処理回路に内建。別層不要。**

### 同一操作の2つの作用範囲
- 快照内「他の疾患に説明済み」= 侧抑制
- 快照間「前回既に説明済み」= 適応
- **抑制対象が違うだけで操作は同じ: 既説明の抑制**

## VeSMed映射: explaining away + 状態記憶

### 方案比較

| | 方案B(独立快照+減法) | 方案C(快照+時間層) | **嗅覚式(EA+記憶)** |
|--|------|------|------|
| 推論引擎 | 不変 | 不変 | explaining away(v3.3) |
| 時間処理 | 手動比較 | 単独後処理層 | **推論回路に内建** |
| 記憶 | なし | 外部 | **状態記憶(顆粒細胞)** |
| 生物対応 | なし | 人工 | **嗅球の精確復刻** |

### 具体動作

```
t0: 患者入院
    evidence = {fever, cough, infiltrate}
    explaining away → 肺炎が認領{fever, cough, infiltrate}
    状態記憶: {fever→肺炎, cough→肺炎, infiltrate→肺炎}

t1: 2日後、新増浮腫+BNP↑
    evidence = {fever, cough, infiltrate, edema, BNP↑}
    状態記憶: fever/cough/infiltrate は肺炎で説明済み
    → {edema, BNP↑}のみ未説明 → explaining awayで処理 → 心不全
    出力: 肺炎(fever,cough,infiltrate) + 心不全(edema,BNP↑)

t2: 抗生素後、熱退+CRP正常化
    肺炎の認領症状が消退 → 肺炎緩解
    心不全の症状は不変 → 心不全持続
```

**合併症が時間で自然分離。後処理不要。**

## 架構的含意: v4.x不要、v3.3+記憶で十分

### 旧ロードマップ
```
v3.3: explaining away（快照内侧抑制）
v4.x: 時系列処理（快照間適応）← 大規模架構変更
```

### 新ロードマップ（嗅覚式）
```
v3.3: explaining away + 状態記憶
      → 快照内侧抑制 AND 快照間適応を同一回路で実現
      → 時系列は別版本ではなく、v3.3の自然な拡張
```

**必要な追加:**
- 状態記憶（前回のexplaining away結果を保持）
- 差分検出（新入力 vs 記憶 → 未説明部分のみ処理）
- CPT/辺/変量は**完全不変**

### 信息=変化（維持）
- 嗅覚系統はdX/dt(一階導数)を符号化、X(絶対値)ではない
- 状態記憶付きexplaining awayは自動的にdX/dtを計算
- 「一直存在の症状は情報を運ばない、変化こそが情報」

## 快照→時系列の進化（維持・修正）

```
現在(快照): {fever: present, CRP: high} → P(D|evidence)
未来(記憶付EA): 
  t1で肺炎を認領 → 記憶
  t2で{edema, BNP↑}追加 → 未説明部分のみ推論 → 心不全追加
  → 疾患組合 + 各疾患の認領症状 + 時間経過
```

## 聴覚との関係: 鸡尾酒会問題

合併症問題の三層分解（修正版）:
| 子問題 | 解法 | 版本 |
|-------|------|------|
| 因果鏈(A→B→症状) | D→D辺 | v3.2 |
| 症状帰属(快照内) | Explaining away | v3.3 |
| 混合信号分離(快照間) | **EA+状態記憶** | **v3.3拡張** |

聴覚の時間相干性（同時変化する音は同一音源）:
- VeSMedでは: 同時変化する症状は同一疾患
- 実装: 状態記憶が「同時に出現した症状群」を自動的にグループ化
- → explaining awayが各群を別疾患に帰属

## 採様層と推理層の分離（維持）

- **採様層**(VeSMed外部): AI音声→文字→構造化データ, EMR自動抽出, センサー
- **推理層**(VeSMed核心): 記憶付きexplaining awayで差分推論
- VeSMedは採様問題を解決しない。データ流を受けて変化を検出するのみ
