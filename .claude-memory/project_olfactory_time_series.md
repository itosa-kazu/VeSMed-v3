## 核心洞察: 信息=变化,不是状态

嗅觉系统编码dX/dt(一阶导数),不是X(绝对值)。Shannon信息量=意外程度=偏离预期。
一直存在的症状不携带信息,变化才携带信息。

## 架构分离(嗅觉受体≠嗅球)

- **采样层**(VeSMed外部): AI语音→文字→结构化数据, EMR自动提取, 传感器。实时/近实时
- **推理层**(VeSMed核心): 接收时间序列,计算导数,做诊断推理
- VeSMed不解决采样问题。假设有人持续喂数据,VeSMed只管从数据流中算变化

## 快照→时间序列的进化

```
现在(快照): {fever: present, WBC: elevated} → P(D|evidence)
未来(时序): [t0:{baseline}, t1:{fever出现}, t2:{CRP跟上}, t3:{热退CRP持续}] → P(D|Δtrajectory)
```

关键优势: "热退但CRP还高"在快照里只是{fever:absent, CRP:high},丢失了"曾经发热然后消退"这个信息。时间序列自然保留。

## 嗅觉三机制在时间序列中自然涌现

| 嗅觉机制 | 时间序列VeSMed | 快照模型能做到? |
|----------|---------------|---------------|
| 适应(背景消除) | 基线帧已存在的→自动降权 | 不能 |
| 侧抑制拓扑 | 同窗口竞争假说互相抑制 | 弱(Noisy-OR隐式) |
| 混合物分离(合并症) | 不同疾病时间演进模式不同→可分离 | 不能 |

## 为什么嗅觉比免疫更贴合

- 嗅觉=侦察兵(识别命名,不负责行动) = VeSMed
- 免疫=执行者(选武器打敌人) ≠ VeSMed
- 嗅觉处理连续信号流 = VeSMed未来
- 免疫处理离散入侵事件 = VeSMed现在(快照)

## Fisher-Rao拓扑 = 嗅球空间地图

paper4_info_geometry.py已实现疾病间Fisher-Rao距离:
- 方法1(CPT-profile): 疾病=高维空间的一团云(每变量一个分布)
- 方法2(后验距离): 疾病=概率单纯形上一个点
- 这就是嗅球的空间拓扑。FR距离近=临床相似=需要强侧抑制
- 比δMed confusion pairs更好: 先验的(从CPT计算),不需要案例
- **拓扑已有,侧抑制待装**

## 嗅觉系统完整机制清单

```
嗅觉受体层（采样）→ VeSMed外部(AI语音/传感器/EMR)

嗅球层（信号处理）:
  1. 适应(Adaptation) → 编码变化dX/dt → 时间序列模型 [版本级方向]
  2. 局部侧抑制 → FR拓扑上近距离疾病互相压制 [有拓扑,待装抑制]
  3. 混合物分离 → 合并症识别(不同疾病时间演进不同) [依赖时间序列]

嗅皮层（模式识别）→ 当前后验计算+排名

主动嗅探(Sniffing) → next-best-test [已有]

未讨论的机制:
  4. 浓度不变性 — 识别模式比例,不是绝对强度 [已分析,见下]
  5. 自上而下调制 — 预期/注意力改变处理("闻煤气"→对煤气更敏感) [待分析]
  6. 模式补全 — 缺几种成分也能认出咖啡 [待分析]
```

## 浓度不变性 = 模式比例识别

嗅觉系统识别受体激活的**比例关系**(2:6:1),不是绝对值(10/30/5 vs 80/240/40)。
机制: 侧抑制归一化掉绝对强度,留下相对对比度。

VeSMed映射: "浓度"=症状严重程度。轻症肺炎和重症肺炎激活变量的模式相同,绝对强度不同。
当前VeSMed对浓度敏感(CRP=mild vs high走不同CPT)→同一疾病轻重版本排名可能不同。
关键场景: 免疫抑制=信号产生能力下降="浓度"降低→又绕回上下文变量。

## 根本结构差异: 嗅觉系统没有疾病节点

VeSMed三件套 vs 嗅觉系统:
- **变量 ≈ 受体** — 大致对应
- **CPT ≈ 结合亲和力** — 勉强对应,但边和CPT在嗅觉中是同一物理实体(分子-受体互补性)
- **边** — 不独立存在,与CPT融合
- **疾病节点** — **完全不存在**。嗅球里没有"咖啡"节点。咖啡=400受体的激活模式本身

VeSMed=二部图(430疾病节点↔900变量节点,8000边)
嗅觉系统=模式空间(每个气味是空间中的一个点/云,识别=最近邻匹配)

**核心发现: VeSMed的"疾病是显式节点"这个架构假设,在嗅觉系统里不成立。嗅觉说:也许不需要疾病节点,模式本身就是诊断。**

**Why:** 从"VeSMed和免疫系统本质区别→感觉系统更贴合→嗅觉适应=编码变化→信息=变化→时间序列"的推理链得出。用户确认方向正确且是版本级方向。
**How to apply:** v4.x或更远的架构规划。当前v3.x仍是快照模型,但设计决策应考虑向时间序列兼容(例如变量设计预留temporal维度)。FR拓扑+侧抑制可能更早实现(v3.x)。
