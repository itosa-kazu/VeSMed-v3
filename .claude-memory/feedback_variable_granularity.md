## 根本原則: 図は臨床観察をモデル化する、生物学機制ではない

D→E辺は全ての中間機制の「净效果」を既にエンコードしている:
```
実際因果鏈: 肺炎 → IL-6↑ → PGE2↑ → 下丘脳体温調定点↑ → 発熱
VeSMedモデル: 肺炎 → 発熱  (1辺で全中間機制を内包)
```

**Why:** 中間変量(IL-6等)の粒度をどこまで加えるか議論。結論: 図に地図の比喩→「地殻プレート運動」は真実だがナビゲーションに不要。

**How to apply:** 新変量追加時に以下の判定基準を適用。

---

## 変量追加の判定基準

> **該検査は目標臨床環境(日本)の標準診断workupの一部か？**

変量集 = 目標臨床環境で実施可能な全検査の**並集**。未実施 = evidence未設定（Noisy-ORが自動無視）。

| 検査 | 標準workup? | 鑑別力? | 判定 |
|------|-----------|---------|------|
| CRP, PCT | ✓ 感染症workup | ✓ 細菌vs非細菌 | **追加** |
| 鉄蛋白 | ✓ HLH/Still workup | ✓ >10000は極めて特異 | **追加** |
| C3/C4 | ✓ 糸球体腎炎workup | ✓ 補体消耗パターン | **追加** |
| LDH | ✓ 溶血/腫瘍workup | ✓ 高値パターン | **追加** |
| IL-6 | ✗ 研究用、臨床常規外 | ✗ 非特異的 | **不追加** |
| TNF-α | ✗ 研究用 | ✗ 非特異的 | **不追加** |

**「機制性」は排除理由にならない。** C3/C4は機制(補体消耗)を反映するが、臨床常規+鑑別力あり→変量として正当。判断基準は常に「臨床で測る+鑑別に役立つ」であり、因果鏈上の位置ではない。

## 評分の二分法: 診断評分 vs 臨床測量尺

| 類型 | 回答する問題 | VeSMedとの関係 | 判定 |
|------|------------|--------------|------|
| **診断評分** (Wells, CURB-65) | "この疾患の確率は？" | VeSMedの**出力**と重複 | **不追加** |
| **臨床測量尺** (GCS, NYHA, Child-Pugh) | "患者の状態は？" | VeSMedの**入力** | **追加（Measurement）** |

- Wells = 組分変量の加権 + 主観判断項(+3)。VeSMedのNoisy-ORが同じ計算をより精確に実行。
- GCS = 意識水準の測定。体温や血圧と同じ臨床状態のMeasurement。

Wells内の「alternative diagnosis less likely(+3分)」は、VeSMedが430疾患の後験比較で天然計算する内容。

**Why:** 当初「複合評分は全て不追加」としたが、GCSのような臨床測量尺まで排除するのは誤り。診断評分(OUTPUT側)と臨床測量尺(INPUT側)は本質が異なる。

## 測量尺の粗細問題: N層嵌套

GCSは実は3層構造:

```
Layer 1 (Finding):         consciousness_impairment: absent / present
Layer 2 (Satellite):       gcs_score: 13-14 / 9-12 / 3-8  (GCS=15 = absent)
Layer 3 (Sub-measurement): gcs_eye / gcs_verbal / gcs_motor
```

Evidence設定規則（全層に再帰適用）:

| 文献報告 | Layer 1 | Layer 2 | Layer 3 |
|---------|---------|---------|---------|
| "意識清楚" | absent | 禁用 | 禁用 |
| "GCS 8" | present(自動) | 3-8 | 未設定 |
| "E2V2M4" | present(自動) | **未設定** | eye=2,verbal=2,motor=4 |

**規則は2条のみ:**
1. parent=absent → 全子孫禁用（Finding→Satellite規則の再帰版）
2. 同層の粗細は互斥（粗 OR 細、両方設定は二重計上）

Finding→Satelliteパターンの自然な泛化。実践上3層を超えることはほぼない。Layer 3は組分に独立鑑別意義がある場合のみ追加。

## 治療反応の二分法: 快照内 vs 快照間

| 時間尺度 | 例 | 框架 | 版本 |
|---------|-----|------|------|
| 秒/分級 | 硝酸甘油、支扩剂 | Finding変量、快照内 | v3.1 |
| 時間/日/週級 | 抗生素退熱、化学療法 | 快照間差分、D→D時間因果 | v3.2 |

境界は物理的: 1回の診察内で観察可能 = 快照内。跨診察 = 快照間。

治療開始後の症状変化で診断を検証するのは、本質的に**快照列の比較**であり、D→D框架の核心応用。

**Why:** 臨床での診断検証の主要手段が「治療後の経過観察」。秒分級は快照内Finding、時間週級は快照間D→D。

## 変量集と疾患集の関係

変量集は疾患集の関数 → 疾患追加時に新変量が必要になりうる。これは**加法演進**であり、螺旋式進歩のfeature。既存変量/辺への影響はゼロ。

技術進歩も同様: 新検査が臨床常規化 → 新変量を加法的に追加。既存構造に影響なし。
