## D→Dは嗅覚に対応なし — GRNが対応

基因調控網絡(GRN): 基因→基因の調控 = D→D辺。

## GRNから抽出したD→D設計規則

| GRN原則 | VeSMed D→D設計 |
|---------|---------------|
| 局部伝播 | 一歩因果のみ、多歩は鏈式涌現 |
| 信号衰減 | 自然衰減（伝播確率の積） |
| 阈値(Hill函数) | explaining awayから涌現（パラメータ不要） |
| 稀疏網絡 | 各疾患2-5条、総計~1000-2000条 |
| Master regulators | 免疫抑制/DM/肝硬変=hub |
| 表観遺伝 | D→D辺もA4 modifier影響下 |

## 阈値: explaining away = Hill函数

```
GRN:    協同結合(n>1) → sigmoid → 二値化
嗅球:   侧抑制 → soft winner → 二値化
VeSMed: explaining away → 競争分配 → posterior鋭化 → 二値化
```
同一操作。追加パラメータ不要。

推論順序:
```
1. Noisy-OR原始評分
2. Explaining away → posterior鋭化（≈Hill函数）
3. D→D伝播（鋭化後posterior使用、追加阈値不要）
4. 再度EA → 収束
```

## GRNのKd進化: 代価-収益均衡

### Kd変化の仕組み
- DNA結合位点突変 → 一条辺のKd変化（局部）
- TF蛋白突変 → 全出辺のKd同時変化（全局）
- 自然選択 → 下流後果の代価-収益で選別

### 各辺のKdは下流後果で独立最適化
| 下流後果 | 進化圧力 | Kd |
|---------|---------|-----|
| 漏激活代価低（熱休克） | 低Kd容忍 | 灵敏（寧可誤報） |
| 漏激活代価高（孢子化） | 高Kd要求 | 保守（確定時のみ） |
| 漏報代価高（DNA修復） | 極低Kd | 極灵敏 |

### VeSMed D→D対応
```
胆管結石→胆管炎: 漏報=脓毒症死亡、誤報=検査のみ → 灵敏(強)
糖尿病→感染:     漏報=中等、誤報=低 → 中等
COPD→肺動脈高圧: 漏報=低(慢性)、誤報=中 → 保守(弱)
```

## D→D辺強度: 暴力調参は禁止

### なぜ進化は暴力試錯できてVeSMedはできないか
- 進化の「テストセット」= 全個体×全世代 = **無限大** → 過拟合不可能
- VeSMedのテストセット = **1268案例** → 1000-2000辺の調参は参数>データ → 過拟合必至

### 鉄則との整合
- feedback_no_overfitting: テスト案例のためだけの変更禁止
- feedback_no_rank_worship: rankは鏡であり目的ではない
- D→D辺強度の暴力最適化 = rank worshipの変種 → **禁止**

### 正しい方法: D→E CPTと同原則
```
辺強度 = 医学知識から（教科書/文献の疫学データ）
回帰テスト = 検証（問題発見）であり最適化ではない
問題発見時 = 医学知識に立ち戻って修正
```

### 唯一最適化可能なもの: 全局超参数
D→D伝播の全体衰減係数等（1-2個）→ 自由度低く1268案例で十分約束 → 過拟合リスク低

### 将来: EHR大数据があれば進化式最適化が可能
- EHR = 数十万〜数百万症例 = 進化の「無限テストセット」に近い
- パラメータ(~2000) << データ(~百万) → 過拟合リスク低
- この時初めて、GRN進化と同様の「試錯→選択→適応」が正当化される
- 現在(1268案例): 医学知識ベース
- 将来(EHR百万): データ駆動最適化

## D→D実装の段階的戦略

```
Phase 1: 拓撲only（有/無辺 + 固定boost）← 教科書常識
  → 回帰テストで検証
  → 不足なら↓

Phase 2: 粗粒度（strong/moderate/weak）← 疫学データ
  → 回帰テストで検証
  → 不足なら↓

Phase 3: 連続精確値 ← 文献のn/N実数計数
  → 大清洗レベルの作業量

Phase ∞: データ駆動最適化 ← EHR大数据
  → 初めて「暴力試錯」が正当化
```
