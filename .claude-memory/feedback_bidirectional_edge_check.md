type: feedback

**辺の漏れチェックは2方向で実施する。**

**方向1: 変量→疾患**
新変量を追加した時に全345疾患と照合。「この疾患がこの変量を直接引き起こすか？」

**方向2: 疾患→変量**
表現が悪い疾患について全200+変量と照合。「この疾患はこの変量に辺があるべきか？」

**Why:** 方向1だけでは、既存変量に対する辺の漏れを検出できない。2026-03-16にS59/S60を追加した際、方向1で17辺の漏れを発見。しかし逆方向（例：D293 BPPVが持つべき辺は何か？）は未実施だった。

**How to apply:**
- 方向1: 新変量追加時に全疾患照合（feedback_new_var_disease_list_check.md）
- 方向2: rank最差の疾患から優先的に、全変量と照合。各変量に対して「この疾患がこの所見を直接引き起こすか？」を判断
- 正常値辺（WBC正常等）は追加不要（feedback_no_normal_value_edges.md）
- 間接因果は直接辺にしない（feedback_no_indirect_edges.md）
