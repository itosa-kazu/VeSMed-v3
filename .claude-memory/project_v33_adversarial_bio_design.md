## 生物から導出した4つの設計原則

### 原則1: Soft Lateral Inhibition（対比増強、非消灭）
生物は全層面(遺伝子→免疫→神経→進化)で**winner-take-allを使わない**。常に概率分布+部分抑制+平行応答。
→ explaining awayは確率加重soft claiming。硬認領禁止。

### 原則2: Additive vs Exclusive所見属性
生物信号は天然に2種:
- **叠加型**(fever, hypoxia, pain): 多因同時産生可能。explaining away不適用
- **排他型**(腫瘍クローン性、特定皮疹形態): 単一起源。explaining away適用
→ 各E変量にadditive/exclusive属性を付与。

### 原則3: Unexplained Residual検出器（先天免疫）
適応免疫(特異的疾患モデル)で識別不能時、先天免疫(非特異的報警)が作動。
→ 最佳疾患組合でも未解釈所見が多い場合「モデル外疾患を考慮」と警告。

### 原則4: D→Dは標準BNの恢復
D→D辺は新数学不要。標準BN belief propagationで処理。D→D=条件先験修正、D→E=Noisy-OR。完全兼容。

## 核心結論
**v3.3に致命的欠陥なし。** 8攻撃全てに生物系統の解法が存在し、VeSMedに移植可能。最も重要な実装決定: **soft claiming（概率加重認領）**。これにより攻撃1,3,4が同時に緩和。

**Why:** v3.3フレームワークの対抗性検証。生物系統が数億年の進化で到達した解を参照。
**How to apply:** v3.3実装時、上記4原則を設計指針とする。特にsoft claimingは最重要。
