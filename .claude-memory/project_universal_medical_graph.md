## 核心洞察（2026-04-05 対話で発見）

### 根本問題の発見
VeSMedの合併症問題を追求していく過程で、より根本的な問題を発見：
- **VeSMedは「匹配(matching)」をしている。臨床が必要としているのは「解釈(explanation)」。**
- 匹配 = 各疾患を個別にスコアリング → 排名表
- 解釈 = 全所見を説明する最良の疾患組合せ → 因果的ストーリー

### 万有医学因果図
全ての医学知識は一枚の巨大な因果図で表現できる：

```
無（健康）──→ ··· 全医学因果図 ··· ──→ 無（死亡）
```

- **図本身** = 医学知識（ノード+辺、静的、全患者共通）
- **患者状態** = 図上のどのノードが活性化しているか（動的、患者固有）

### ノードの種類
全て同一の図上に共存。辺の意味は統一的：「AがBの発生確率を増加させる」

| ノード種 | 例 | 計算上の役割 |
|---------|-----|------------|
| 危険因子/行動 | 喫煙、肥満、遺伝 | 既知の入力 |
| 疾患 | COPD、肺癌、肝硬変 | 推論対象（D→D辺で相互接続）|
| 合併症/病態 | 門脈圧亢進、DIC | 疾患の下流ノード（D→D辺）|
| 所見 | 咳嗽、黄疸、WBC↑ | 観察可能な出力 |

### 二種の辺（快照内 vs 快照間）

| 辺の種類 | 意味 | 例 | 快照 |
|---------|------|-----|------|
| **D→E（瞬時因果）** | 疾患が所見を生む | 肺炎→発熱 | 快照**内** |
| **D→D（時間因果）** | 疾患が合併症を引き起こす | COPD→肺炎 | 快照**間** |

快照内はD→Eのみ → **DAG保証**（BN推論適用可）
快照間にD→D → 環路許容（vicious cycleは時間展開で解消）

### 全医学行為の統一表現

| 概念 | 図上の意義 |
|------|-----------|
| 診断 | 観察された路標（所見）から**活性ノードを推定** |
| 治療 | ノードを**削弱**（水泵の比喩：自然方向は健康→死亡、治療は逆向きの泵） |
| 予後 | 現在の活性ノードの**下流経路を予測** |
| 予防 | 危険経路への**進入を阻止** |
| 時間 | 活性ノード集合の**快照列** |
| 分期 | 活性ノードの**state確定**（診断と同一の情報理論的操作） |
| 監測 | 過去の快照との**差分検出**（stateが変化したか） |

### 水流の比喩（診断の物理的直感）

- 所見（観測）= 水源。各所見が水を注ぎ込む
- 辺 = 水路。因果関係が水の流れを導く
- 疾患ノード = 盆地。水が自然に集まる場所 = 診断
- 一つの盆地に全水が集まる → 単一疾患の説明
- 複数の盆地に分かれる → 多疾患の説明（図の地形が自然に決定、parsimony不要）
- 治療 = 泵（盆地から水を汲み上げ、健康方向へ押し戻す）

### 合併症問題の自然消滅

合併症は「特殊問題」ではない。図上の隣接ノードに過ぎない：
```
胆管結石(D) → 胆管炎(D) → 発熱(E)
```
D→D辺を追加するだけで、合併症は図の自然な一部となる。

## 検査推奨の統一理論

診断/分期/監測は全て**同一の情報理論的操作**：

> **「図上のどこがまだ不確実か？何を観測すればそれが明確になるか？」**

| 臨床段階 | 不確実性の対象 | 推奨操作 |
|---------|-------------|---------|
| 診断 | どのノードが活性か？ | 鑑別検査（ノード特定） |
| 分期 | 活性ノードのstateは？ | 分期検査（state特定） |
| 監測 | stateが変化したか？ | 経過観察検査（差分検出） |

数学的には全て: argmax_test E[ΔH] = **次に観測すべき変数は、不確実性を最も減少させるもの**

### Stageは新変量不要
- Stage = 複数の既存変量の組合せから**涌現**する結論
- 例: 肺癌stage = L_tumor_size + L_lymph_involvement + L_distant_metastasis
- 各変量は通常のFinding/Measurement → staging CTはこれらを同時にinform
- 推薦ロジック: 活性ノードに接続する未観測変量のうち最大ΔHのものを推薦

## 対抗検証記録

### 第1回（概念レベル 5攻撃）

| # | 攻撃 | 判定 | 解決法 |
|---|------|------|--------|
| 1 | 環路(vicious cycle) | **解決** | 快照内DAG + 快照間で時間展開 |
| 2 | 非純因果 | **帰着** | 攻撃1に帰着（双方向=環路、共因=図の完備性） |
| 3 | 健康/死亡の非対称性 | **解決** | 水泵比喩（自然方向=下坡、治療=逆向泵、自然治癒=内蔵泵） |
| 4 | ノード粒度 | **解決** | 因果profile異→別ノード、重症度異→同ノード別state（Domain Invariant準拠） |
| 5 | 規模 | **受容** | 必然的に大。工程問題 |

### 第2回（落地レベル 5攻撃）

| # | 攻撃 | 判定 | 解決法 |
|---|------|------|--------|
| 6 | 多層推論の計算量 | **撤回** | 430ノード×10辺×5iteration=微秒級。10,000ノードでもミリ秒 |
| 7 | D→D辺のCPTデータ源 | **低リスク** | D→Dは粗粒度(strong/moderate/weak)で十分。LLM+教科書常識で生成可。PMIDは不要 |
| 8 | 疾患stateが変量体系にない | **解決** | stageは既存変量の組合せから涌現。新変量型不要 |
| 9 | 最小覆盖≠最佳解釈 | **解決** | 水流比喩：地形（図の構造）が自然に決定。人為的parsimony不要 |
| 10 | 快照内に環路あり？ | **解決** | 因果は時間を要する。瞬時にはD→E(DAG)のみ。D→Dは時間因果→快照間 |

**結論: 2回10攻撃を全て化解。フレームワークはrobust。**

## VeSMedとの関係と実装ロードマップ

**VeSMedは既にこの図の部分グラフ。唯一の拓撲変化はD→D辺の追加のみ。**

| 組分 | v3.0（現行） | 万有医学因果図 | 変化 |
|------|------------|--------------|------|
| D節点 | 430疾患 | 同 | 無 |
| E節点 | 901変量 | 同 | 無 |
| R→D辺 | 年齢/性別prior | 同 | 無 |
| D→E辺 | 8201条 | 同 | 無 |
| **D→D辺** | **無** | **新規** | **唯一の拓撲変化** |
| 推論引擎 | Noisy-OR単pass | +D→D伝播pass | 小改 |

最深の洞察が最小の架構変化を導く = 優雅の定義。理論がVeSMedの推翻を要求しない = 理論が正しい証拠。

### 変量設計規則（方法論、コード変更なし）
- 三層構造（Finding/Measurement/Satellite）+ N層嵌套
- 変量粒度原則（臨床観察のみ、診断評分≠臨床測量尺）
- 2条の再帰規則（parent=absent→子孫禁用、粗細互斥）
- 全E節点はグラフ上扁平、層級関係はevidence設定規則（UI層）で管理

### 推論の変更点

```
Pass 1: E→D推論（現行のNoisy-OR、そのまま）
Pass 2: D→D伝播（活性疾患が下流疾患をブースト）
繰り返し収束まで（3-5回、微秒級）
```

### ロードマップ

- **v3.1（大清洗）**: D→E辺の品質確保(UpToDate+文献) + D→D辺を収集（粗粒度strong/moderate/weak、UpToDate審査時に同時記録）+ 変量設計規則の適用
- **v3.2**: D→D推論実装 + 検査推奨の統一化（診断/分期/監測を同一ΔHフレームワークで統合）
- **v3.3**: 水流比喩の出力UI（排名表→因果図表示）
- **論文化**: 万有医学因果図の理論 + 40年間の合併症問題への解答 + VeSMedでの実証

## 発見の経緯

1. 合併症問題の検討 → 方案B(提示層)+C(疾患境界再定義)は「不優雅」
2. 文献調査 → 40年間未解決（DXplainは"impossible"、PCTはNP-complete）
3. 「臨床医はそんな複雑に考えていない」→ 根本に立ち返る
4. 「VeSMedのrank 1は誰に？」→ 単一rank自体が間違い
5. 「なぜ診断する？治療のため」→ 診断は手段、目的は行動
6. Q1-Q5は全て同一対象の異なる視点 → 「場景(scenario)」概念
7. 「場景」→「説明」→「因果図」→ 全医学は一枚の図
8. 治療=ノード削弱(水泵)、時間=快照列 → 全統一
9. 「也许只有一个开始和一个终点？無と無」→ 万有医学因果図
10. 肺癌のstage問題 → 検査推奨が診断/分期/監測を統一する
11. 対抗検証2回10攻撃 → 全て化解、フレームワーク確立

**Why:** 合併症の「不優雅な解決策」に対するユーザーの不満から出発。個別の問題を解こうとするのではなく、根本問題（匹配vs解釈）を特定することで全問題が自然消滅する洞察に到達。

**How to apply:** v3.1で大清洗（D→E品質確保 + D→D収集）。v3.2でD→D推論。v3.3でUI刷新。論文化可能性大（40年未解決問題への新アプローチ）。
