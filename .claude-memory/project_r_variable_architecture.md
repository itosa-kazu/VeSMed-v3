## R変数の仕組み（2026-03-18確認）

### 二つの経路
| | S/E/L/T変数 | R変数 |
|--|-------------|-------|
| step2辺 | あり（D→変数） | **なし** |
| step3 CPT | `noisy_or_params` | `full_cpts` |
| 方向 | 病→所見 P(所見\|病) | 人→病 P(病\|背景) |
| 臨床的意味 | 病気になったら何が出るか | 症状を見る前の起跑線 |

### prior_power bug (修正済み)
- `app.py` が `infer()` に `prior_power` を渡していなかった（デフォルト0.0）
- R01/R02のfull_cpts（176+61疾患分）が**初日から完全に死コード**
- 修正で **+26 Top-1** (69.9%→73.2%)
- `next_best_test()` / `next_best_falsification_test()` も同様に修正済み

### アーキテクチャ変更（2026-03-18）
- **独立乗法LR** (DeepMind三層BN, Richens et al. 2020)
- 各R変数が独立にLR（尤度比）を貢献 → log空間で加算
- 旧: 結合CPT（R01×R02で14エントリ）→ 新: per-R個別CPT（7+2エントリ）
- `full_cpts` で新旧フォーマット共存:
  - 旧: `{"parents": ["R01","R11"], "cpt": {"18_39|no": 0.06, ...}}`
  - 新: `{"parents": ["R11"], "cpt": {...}, "R01": {"18_39": 0.01, ...}}`
- 結果: Top-1 +12 (612→624)
