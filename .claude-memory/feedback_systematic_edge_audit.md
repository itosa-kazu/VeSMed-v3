疾患追加後は必ず**体系的辺監査**を実施する。

**方法:**
1. 全案例のrankを計算
2. rank≥2の案例について、expected_diseaseのedge集合とevidence変数集合を比較
3. evidence変数にedgeがない（=漏れ辺）ペアを全て列挙
4. 臨床的に妥当なものだけ追加（特殊変量やT01/T02の欠落はスキップ）

**実績(2026-03-15):**
- rank2-3の67件 + rank4-5の16件から**144本の漏れ辺**を検出
- うち臨床的に妥当な**73本**を追加（27+42+α）
- 結果: Top-1 285→321(+36!), Top-3 383→395(+12!), 77%/95%達成

**Why:** 疾患追加時に辺を最小限しか定義しないため、実案例のevidence変数にedgeがない状態が頻発。edgeがない変数はleak値で評価され、期待疾患が不利になる。

**重要教訓(2026-03-15):** 新疾患追加後のTop-1低下を「IDF影響」と安易に帰結するな。実際には**漏れ辺が主原因**であることがほとんど。211疾患時に「IDF影響で-3」と報告したが、辺監査で21本追加したら+8回復した。IDF飽和は確かに存在するが、辺監査で修復可能な問題を先に排除すること。
**How to apply:** 10疾患追加ごと、またはTop-3が低下した時に、以下のスクリプトで監査:
```python
# evidence変数にedgeがない(disease,variable)ペアを列挙
missing = set(ev.keys()) - edge_map.get(expected, set())
```
臨床的に妥当な辺のみ追加。根拠不明な辺は追加禁止（CPT文献鉄則に準ずる）。
