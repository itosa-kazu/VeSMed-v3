## 方針
システム架構は疾病無関。BN推論エンジン(Noisy-OR+IDF+CF+反証推奨)はそのまま使える。
疾患・変量・辺を追加するだけで拡張可能。

## 拡張ロードマップ

### Phase 1: 呼吸困難鑑別 ← 着手中
未追加12疾患:
  D120 急性心不全, D121 COPD急性増悪, D122 喘息発作, D123 気胸,
  D124 心タンポナーデ, D125 ARDS, D126 上気道閉塞, D127 不整脈,
  D128 ILD, D129 胸水(非感染性), D130 アナフィラキシー, D131 過換気症候群
新変量6個:
  L51 BNP, S49 起座呼吸, E36 下腿浮腫, E37 JVD, R44 心疾患既往, R45 喫煙歴

### Phase 2: 胸痛鑑別
ACS(急性冠症候群)、大動脈解離、気胸、GERD、肋骨骨折 等
（PE/心筋炎/心膜炎は既存）

### Phase 3: 腹痛鑑別
虫垂炎/胆嚢炎/膵炎/腸閉塞/消化管穿孔 等
（大部分が既存D10/D11/D86等、追加は少ない）

### Phase 4: 意識障害鑑別
脳卒中、低血糖、薬物中毒、てんかん重積 等
（髄膜炎/脳膿瘍/脳炎は既存）

### Phase 5: 全部統合
**Prior調整が必要**: 現在のpriorは発熱門診の患者集団を前提(BASE=0.01)。
全疾患に拡張する場合、一般救急外来の有病率分布に合わせてBASEとfull_cptsのprior値を再調整する必要がある。
拡張初期は発熱用priorのままでも動くが、最終的には人口ベースの有病率データで再calibrationが必要。

## 不漏のための系統的アプローチ
1. 教科書/ガイドラインの鑑別リストを参照
2. ER見逃し率Top-15等のエビデンスベースリスト
3. 臓器系統別に網羅確認
4. 追加後に既存案例の回帰テスト必須
