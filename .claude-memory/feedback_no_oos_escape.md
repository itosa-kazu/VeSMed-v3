name: OOS逃避禁止 — FATALを正直に受け入れる
description: 診断困難な案例をOOSにして隠蔽するのは禁止。正面から変数追加で解決するか、FATALとして受け入れる。
type: feedback
---

診断困難な案例をOOSにマークして問題を隠蔽するのは**禁止**。

**正しい対応:**
1. 新変数を追加して鑑別力を改善する（例：クレアチニン変数でTTP/HUS鑑別）
2. それでも解決しない場合は、FATALとして正直に受け入れ、将来の課題として記録する

**Why:** R348(非典型HUS)がTTPと鑑別できずFATALになった際、OOSにマークして逃避しようとした。ユーザーが「把R348标OOS就是逃避问题」と指摘。成績を良く見せるための操作は証拠操作と同質。
**How to apply:** FATALが出たら、まず変数/辺の追加で解決を試みる。それでもダメならFATALとして残し、必要な新変数（例：ADAMTS13）を将来課題に記録する。OOS/in_scope=falseは真にスコープ外の案例のみに使用。
