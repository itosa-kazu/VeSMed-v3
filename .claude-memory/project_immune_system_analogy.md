## 核心洞察: 先天免疫的真正作用是**信息定性**,不是粗筛

免疫系统两层架构不是为了省算力(现代计算机不需要)。先天免疫的核心功能是:
- 检测到PAMP → 分泌特定细胞因子 → **极化**适应性免疫的响应方向(Th1/Th2/Th17)
- 同一个抗原,在不同细胞因子上下文中,适应性免疫的解读完全不同

## VeSMed映射

**现状**: 所有变量通过Noisy-OR独立贡献到疾病概率,变量之间无交互。

**免疫系统启发**: 某些变量的作用不是直接指向疾病,而是**改变其他变量的解读方式**。

例: CRP=elevated 在说"这是炎症框架" → ferritin_very_high 在炎症框架下指向HLH/成人Still,在非炎症框架下指向血色素沉着症。但当前Noisy-OR中CRP和ferritin是独立贡献的。

## 架构含义

- Finding/Satellite三层设计已有这个影子(Finding定性→Satellite细化),但仅限同一变量族内部
- 免疫系统告诉我们:**这个机制应该跨变量起作用**(CRP的状态影响ferritin的解读)
- 实现需要V→V辺(变量间条件依赖),超出Noisy-OR的表达力 → v3.x架构级方向

## 杀手级应用: 免疫抑制状态问题

免疫抑制(HIV/移植/化疗等)是最明显的"上下文变量":
- 同一所见在正常vs免疫抑制下CPT完全不同（肺浸润→普通肺炎 vs PCP）
- 当前Noisy-OR中免疫抑制只是独立贡献的一条边，不改变其他变量的解读
- WBC正常在免疫抑制下不排除感染，但固定CPT会把感染往下拉 → 两个信号打架

其他上下文变量: 妊娠、年龄极端值、慢性肝病/肾病、糖尿病

## 与中间变量的关系

**不打架，是正交的两个概念:**
- **中间变量**(L63-L67): 解决 D→[隐藏中间状态]→V 的间接因果链。方式=把中间状态变成可观测的lab/sign变量
- **上下文变量**: 解决"同一证据在不同上下文中含义不同"。方式=V→V辺,让上下文变量调节其他变量→D的CPT

两者可以共存: 例如免疫抑制(上下文)可以同时改变L64(DIC评分,中间变量)→D的CPT解读

## 细胞因子风暴=全激活=侧抑制崩溃

Th1/Th2/Th17/Treg互相拮抗是必须的——杀胞内菌(Th1)和驱寄生虫(Th2)的策略物理上矛盾。
全激活=细胞因子风暴=多器官衰竭。没有"选择"就没有有效应答。

VeSMed可能需要**两层侧抑制**:
1. 框架层(感染vs自身免疫vs肿瘤) — 对应Th极化
2. 疾病层(肺炎球菌vs金葡) — 对应克隆竞争

## 已解决的问题

1. **VeSMed不一定需要Th1/Th2式拮抗。** 免疫系统的拮抗源于物理资源冲突(一个T细胞只能分化一次)。VeSMed是数学系统，无此约束。侧抑制/explaining away的理由应从VeSMed自身问题出发，不照搬。
2. **为什么一个IL对应复数病原体？→ 核心启发: 按"武器"降维，不按"敌人"降维。**

## 降维启发: 按应对策略分类

免疫系统面对的核心问题: 新敌人无穷无尽，但有效杀伤机制只有几种。
进化的解法: 不按病原体(无限维)分类，按应对策略(4维: Th1/Th2/Th17/Treg)分类。

VeSMed映射:
- 现在: 症状 → 430种疾病(按敌人分类)
- 免疫系统思路: 症状 → 少数"临床框架"(按应对策略) → 框架内精确鉴别
  - "需要抗生素的" ≈ Th1/Th17 → 细菌感染内部鉴别
  - "需要免疫抑制的" ≈ Treg → 自身免疫内部鉴别
  - "需要手术的" → 外科疾病内部鉴别
  - "需要化疗的" → 恶性肿瘤内部鉴别

关键洞察: 中间层分类标准应是**治疗学的**(怎么对付)，不是病因学的(敌人是谁)。治疗策略种类远少于疾病种类。这是降维问题，不是拮抗问题。

## 类比的边界: 治疗策略降维在人类医学中漏水

- 免疫系统: 4种武器(Th1/Th2/Th17/Treg)，**互斥**(一个T细胞只能选一个方向)。互斥=强降维
- 人类医学: 7-10种基本策略(抗微生物/免疫抑制/手术/化疗/替代/抗凝/支持)，**可叠加**不互斥
  - 感染性心内膜炎=抗生素+手术; SLE合并感染=免疫抑制+抗生素(直接矛盾!)
- 可叠加的标签系统 ≠ 真正的降维。缩小搜索范围有用，但不如互斥分类干净
- **结论**: "按武器降维"的启发有价值但在VeSMed不能直接复制。也许在别的地方能用到
- 这也说明VeSMed和免疫系统干的事有本质区别:

## 本质区别: 执行者 vs 侦察兵

- **免疫系统=执行者**: 输入分子模式→输出效应行动(杀伤/中和/耐受)。不需要知道敌人叫什么，只需知道怎么打
- **VeSMed=侦察兵**: 输入临床所见→输出疾病名称+概率。只需知道敌人叫什么，怎么打不归它管

这个区别解释了所有类比漏水之处:
- 互斥决策: 执行者必须选一个方向(一个细胞只能分化一次)，侦察兵只需报告概率分布
- 按武器分类: 执行者的核心需求(选武器)，侦察兵不需要(目的是命名)
- 速度压力: 执行者致命(小时级)，侦察兵可以等检查结果

**大自然里的"侦察兵"是感觉系统**(视觉/嗅觉/听觉)——只负责识别命名，把结果交给运动/决策系统。嗅觉类比比免疫类比更贴合VeSMed的本质。

免疫类比的有效部分: ①上下文变量改变CPT解读(与执行/侦察无关，依然成立) ②按策略降维+两层架构(在VeSMed里漏水，需从自身问题重新推导)

**Why:** 从免疫系统两层架构(先天/适应性)的深度类比中得出。免疫抑制=先天免疫被破坏,完美证明上下文定性的重要性。
**How to apply:** 当讨论架构演进、变量交互、CPT上下文依赖、免疫抑制患者表现差等话题时,参考此类比。这是一个尚未实现的研究方向。
