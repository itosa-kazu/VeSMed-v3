## 各攻撃の詳細

### 攻撃1: EA初期化依存性（⭐⭐⭐⭐）
**問題**: EAは逐次的にcredit assignmentを行う。最初にどの疾患が症状を「認領」するかで、後続の疾患の評価が変わる。処理順序が結果を左右する。
**嗅覚との対比**: 嗅覚の侧抑制は**並列**（全神経元が同時に競争）。VeSMedのEAが**逐次**なら、嗅覚と非同構。
**鍵問題**: 並列競争をどう実装するか？

### 攻撃2: 記憶誤差鎖死（⭐⭐⭐⭐⭐）最危険
**問題**: 快照1で疾患Aが症状Xを誤って認領→記憶に格納→快照2で症状Xは「既解釈」として無視→真の原因Bが永遠に発見されない。誤差が自己強化するpositive feedback loop。
**生物対照**: 免疫記憶にもautoimmune diseaseという同様の失敗モードがある。
**鍵問題**: 記憶の「忘却」メカニズムが必要？

### 攻撃3: EA↔D→D反復順序（⭐⭐⭐）
**問題**: EAの結果がD→D伝播の入力になり、D→Dの結果がEAの入力になる。どちらを先に計算するかで結果が変わる。収束保証なし。
**対策候補**: 交互反復で収束を待つ（loopy BP的アプローチ）

### 攻撃4: EAが真の合併症を殺す（⭐⭐⭐⭐）
**問題**: 合併症は本物の複数疾患が共存。EAの侧抑制が強すぎると2番目の疾患を消す。弱すぎるとEAの意味がない。
**第1轮との関連**: 第1轮攻撃1と同根。soft claimingで緩和だが、softの程度の調整が新たな問題。

### 攻撃5: 計算量爆発（⭐⭐⭐）
**問題**: 430疾患×430疾患のD→D + 900変量のEA + 反復収束 = 1案例の推論に数秒〜数十秒？リアルタイム臨床使用に支障。
**対策候補**: 上位N疾患のみでEA実行（pruning）、D→Dは疎行列

### 攻撃6: 「同一回路」仮説の脆弱性（⭐⭐⭐）
**問題**: 快照内の侧抑制（競争分配）と快照間の適応（背景消去）は、概念的には「既解釈の抑制」で統一できるが、実装上は異なるデータ構造（即時計算 vs 永続記憶）が必要。「同一回路」は比喩に留まる可能性。
**対策候補**: 操作は同一（subtract explained）、データ構造は異なる、で許容

### 攻撃7: IDF削除による性能退行（⭐⭐⭐）
**問題**: IDF(dp=0.7)は現行v3.0のTop-1 78%に貢献。EAが完成する前にIDFを削除すると性能崩壊。しかしIDF+EA併用は理論的に二重計算。
**対策候補**: 段階的移行。IDF→EA置換時に性能が維持されることを確認してからIDF削除。

## 優先順位

3. **攻撃4**(合併症殺害): soft claimingの程度調整
4. **攻撃3,5,6,7**: 実装段階で対処可能
