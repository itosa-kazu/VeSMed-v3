## 各攻撃の詳細

### 攻撃1: EA初期化依存性（⭐⭐⭐⭐→⭐ 解決済）
**問題**: EAは逐次的にcredit assignmentを行う。最初にどの疾患が症状を「認領」するかで、後続の疾患の評価が変わる。処理順序が結果を左右する。
**嗅覚との対比**: 嗅覚の侧抑制は**並列**（全神経元が同時に競争）。VeSMedのEAが**逐次**なら、嗅覚と非同構。
**解法**: **Jacobi同期更新**。全疾患が「前轮の快照」を基に新claim値を計算→全部計算完了後に同期更新。数学的に並列競争と等価。収束保証はsoft claiming(∈(0,1))による圧縮映射+Banach不動点定理。
**先行事例**: QMR-DT(Shwe 1991)が同様のBN推論を試みたが当時は計算不可能。現在の430疾患規模では実用的。

### 攻撃2: 記憶誤差鎖死（⭐⭐⭐⭐⭐→⭐ 解決済 2026-04-06）
**元の問題**: 快照1で疾患Aが症状Xを誤って認領→記憶に格納→快照2で症状Xは「既解釈」として無視→真の原因Bが永遠に発見されない。
**根本原因の発見**: 問題は「記憶の忘却が足りない」ではなく、**記憶の抽象レベルが間違っていた**。
**嗅覚の答え**: 嗅覚の記憶は**信号級**（受体17=0.8, 受体42=0.3...）。「この匂いは珈琲」という**解釈を記憶しない**。
**VeSMed解法 — 信号級記憶の原則**:
- 記憶に格納: `{発熱=present, 血糖=high, ...}` ← 信号そのもの（what I saw）
- 記憶に格納しない: `{発熱→肺炎}` ← 解釈（what I think it means）
- 記憶に格納しない: `{D→D:糖尿病→感染已激活}` ← 推理状態
- 毎快照、EA+D→Dは**全て再計算**。記憶はEAの入力重みにのみ影響（旧信号の適応/衰減）
**効果**: 解釈を記憶しないので鎖死が原理的に不可能。回顧性重評估も不要（毎回全計算なので）。
**三安全弁は衰減のみに簡化**: 記憶=信号+衰減weight。soft/overrideはEA毎回再計算に内包。
**設計格言**: 「記憶は"何を見たか"であり"何だと思ったか"ではない」

### 攻撃3: EA↔D→D反復順序（⭐⭐⭐→⭐ 解決済 2026-04-06）
**問題**: EA→D→D? D→D→EA? 鶏卵問題。互いに相手の出力を必要とする。
**解法**: 攻撃1と同一。Jacobi同期迭代で両者を同時更新。攻撃1がEA内部の順序問題、攻撃3はEA⇔D→D間の順序問題。スケールが違うだけで本質は同じ→同じ解法で一括解決。

### 攻撃4: EAが真の合併症を殺す（⭐⭐⭐⭐→⭐ 解決済 2026-04-06）
**問題**: 真の合併症(糖尿病+肺炎等)で共有症状をEAが競争分配→2番目の疾患を殺す。
**嗅覚の答え**: 侧抑制は**対比増強**(contrast enhancement)であり**赢者通吃**(winner-take-all)ではない。両方生き残る、差が広がるだけ。
**解法**: soft claiming = 標準Bayes概率分配。claim(D,E) = P(D)×P(E|D) / Σ P(D')×P(E|D')。新発明不要、Bayes公式そのもの。
- 単一疾患のみ解釈可能 → ≈100%認領（赢者通吃的）
- 複数疾患が解釈可能 → 概率比例で分配（対比増強的）
- 合併症は殺されない、份額が小さくなるだけ
**残課題**: 叠加型症状(additive: 発熱等、複数疾患で重症化)のEA除外。議論中

### 攻撃5: 計算量爆発（⭐⭐⭐）
**問題**: 430疾患×430疾患のD→D + 900変量のEA + 反復収束 = 1案例の推論に数秒〜数十秒？リアルタイム臨床使用に支障。
**対策候補**: 上位N疾患のみでEA実行（pruning）、D→Dは疎行列

### 攻撃6: 「同一回路」仮説の脆弱性（⭐⭐⭐）
**問題**: 快照内の侧抑制（競争分配）と快照間の適応（背景消去）は、概念的には「既解釈の抑制」で統一できるが、実装上は異なるデータ構造（即時計算 vs 永続記憶）が必要。「同一回路」は比喩に留まる可能性。
**対策候補**: 操作は同一（subtract explained）、データ構造は異なる、で許容

### 攻撃7: IDF削除による性能退行（⭐⭐⭐）
**問題**: IDF(dp=0.7)は現行v3.0のTop-1 78%に貢献。EAが完成する前にIDFを削除すると性能崩壊。しかしIDF+EA併用は理論的に二重計算。
**対策候補**: 段階的移行。IDF→EA置換時に性能が維持されることを確認してからIDF削除。
