## 専家の推理 = explaining away + 記憶

### 査房の第一問: 「昨天到今天有什么変化？」
```
出力 = 新入力 − 記憶 = 変化量
```
嗅球顆粒細胞の減算と完全同一。

### 具体過程
```
入院Day1: 発熱+咳嗽+浸潤影 → 「肺炎」認領 → 記憶に格納
Day2査房: 「什么新的？」→ 浮腫+BNP↑(新発) → 「心不全」追加
  ※ 発熱/咳嗽は再評価しない（既に解釈済み=適応済み）
Day3査房: 熱退+CRP↓ → 「肺炎緩解」。浮腫不変 → 「心不全持続」
```

## 新手 vs 専家 = Noisy-OR vs EA+記憶

| | 新手 | 専家 |
|--|------|------|
| 毎回の評価 | 全症状を最初から再評価（快照重算） | **変化のみ処理**（差分） |
| 共有症状 | 全疾患に同等の信用付与（競争なし） | **自動無視**（既解釈=既適応） |
| 不適合症状 | 見落とし可能 | **即座に警戒**（unexplained residual） |
| 合併症 | 分離困難 | **時間順で自然分離** |
| VeSMed対応 | Noisy-OR (v3.0) | EA+記憶 (v3.3) |

**新手→専家の成長 = Noisy-OR→explaining away+記憶の進化**

## 違和感メカニズムの解明

資深医の「何かおかしい」= **unexplained residual検出器**
- explaining awayで全症状を分配完了
- 残余（どの疾患にも認領されない症状）が存在
- → 「まだ何か説明できていない」→ 追加検査
- 新手はexplaining awayをしていないので残余を検出できない

## 収束進化: 嗅覚と専家推理が同一解に到達

**同一問題**: 混合信号から原因を識別する
**同一解**: 組合符号化 + 競争的credit assignment + 状態記憶 + 残余検出

嗅覚系統: 数億年の進化で到達
専家臨床推理: 数千年の医学+個人の臨床経験で到達
VeSMed v3.3: 上記二者からの逆設計で到達

**How to apply:** v3.3の設計検証（専家推理と同構ならば正しい方向）。論文のnarrative素材。
