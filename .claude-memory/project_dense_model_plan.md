## 実験結果（2026-03-16）

### 試したこと
1. 全171変量×345疾患をleak値でauto-fill → 59162辺
2. IDF公式をcpt_variance(CPT方差)に変更
3. cf_alphaをlog-LRベース(辺の有無→log-LR>0.1)に変更

### 結果
| モデル | Top-1 | Top-3 |
|--------|-------|-------|
| **main(稀疏+旧IDF+旧cf_alpha)** | **519** | **625** |
| 全稠密+新IDF+新cf_alpha | 497 | 611 |
| pure log-LR(IDF/cf_alpha無し) | 483 | 600 |

### 失敗の原因
- leak auto-fill(CPT=leak)はlog-LR=0 → 辺がないのと完全に同じ → 意味がない
- 稠密化は「全疾患に辺がある」ことが目的ではなく、「全疾患に臨床的に正確なCPTがある」ことが目的
- BPPV→Cr正常95%（臨床判断）≠ leak値80%（背景確率）→ この差がないと鑑別力にならない

### 正しいアプローチ（確定）
**leak auto-fillではなく、逐病逐変量の臨床判断CPT設定。**
- 各疾患×各変量について「この病はこの検査結果にどう影響するか」を臨床的に判断
- 影響する → 具体的CPT値を設定
- 影響しない → 「正常値確率が高い」CPTを設定（leakとは異なる！）
- これを345×172=59340辺分やれば真の稠密モデルになる

### 到達時の計画
- 全辺が臨床CPTで埋まったら、IDF公式をcpt_varianceに切替え
- cf_alphaもlog-LRベースに切替え
- grid searchで再最適化

### 現実的な進め方
- 現在は稀疏モデルのまま最適化を続ける
- 逐病逐変量の臨床判断CPT設定を少しずつ進める（edge-audit Step6）
- 「正常値辺」もleak値ではなく臨床判断で設定する（例：BPPV→Cr正常95%）
- 目標はあくまで59340辺の臨床CPT完成だが、年単位の作業

### 教訓
- **leak auto-fillは捷径に見えて無意味**（log-LR=0）
- **旧IDF/cf_alphaは稀疏モデルでは優秀**（辺の有無がモデル充実度のproxy）
- **稠密化の前にまず稀疏モデルの辺を充実させるのが先**
