**新疾患を追加したら、テスト案例検索は絶対に省略してはならない。**

これは鉄則6「新疾患追加時テスト案例必須」の再確認であり、最重要レベルの教訓。

**インシデント(2026-03-15):**
D197-D251の55疾患を連続追加した際、「5個ずつ高速で追加→回帰テスト→commit push」のサイクルに没頭し、PMCからのreal case検索を完全に忘れた。ユーザーに「新的病，你找real case测试了吗？」と指摘されて発覚。

**根本原因:** 疾患モデル追加の速度を優先し、テスト案例検索プロセスをスキップした。回帰テストだけでは新疾患の動作確認にならない（回帰は既存案例しかテストしない）。

**対策（強制）:**
1. 疾患モデル追加スクリプト実行後、**必ず**案例検索agentを起動
2. 案例検索→追加→テスト完了するまで次の疾患バッチに進まない
3. 5疾患追加→5疾患分のagent検索→案例追加→回帰テスト→commit の完全サイクルを守る
4. 「あとでまとめて」は禁止。疾患追加と案例追加は同一commitが理想

**Why:** テスト案例なしでは新疾患の推論精度が検証できない。モデルに疾患を追加しただけでは、その疾患が実際にTop-3に入るかどうか不明。55疾患分のテスト不在は品質保証の空白。
**How to apply:** 疾患追加のワークフローに案例検索を組み込み、省略不可能にする。
