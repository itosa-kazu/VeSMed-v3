### IDF = explaining awayの静的近似（v3.3で削除予定）
- IDF: 静的・全局的・確率基礎なし（Noisy-ORの補丁）
- Explaining away: 動的・局部的・Bayes credit assignment
- 数学: IDF ≈ 1/P(E)の粗近似、explaining away = P(E)の精確計算
- v3.3実装時にIDF削除 → explaining awayが自然に代替

### 浓度不変性の分析
- A4(context-dependent CPT)の全局サブセットを自動解決する案だった
- 入力空間の異質性で断念。log-LR+IDFが穷人版として機能
- A4は稀疏修飾層で個別対処（不優雅だが問題構造に正直）

### 形状向量仮説（→ project_nmf_experiment.md）
- CPT矩陣有効次元≈74、NMF因子が器官系統に自動対応 → 形状は存在する
- Netflix式補完は鑑別情報を破壊 → 補完には使えない
- 残差 = 鑑別情報 = IDFの精密版 → QCツールに活用可

## 聴覚から得た架構洞察: 鸡尾酒会問題

### 合併症問題の三層分解
| 子問題 | 解法 | 版本 | 時系列必要? |
|-------|------|------|-----------|
| 因果鏈(A→B→症状) | D→D辺 | v3.2 | 不要 |
| 症状帰属(この症状はどの病?) | Explaining away | v3.3 | 不要 |
| **混合信号分離(重疊症状の分離)** | **時間相干性** | **v4.x** | **必要** |

### 時間相干性 = 合併症分離の最終解
快照では肺炎+心不全の共有症状(呼吸困難)を分離不可。
時系列では:
```
t0: 基線（心不全安定）
t1: 発熱+咳嗽が同時出現 → 同時変化 → 同一疾患(肺炎)
t2: 呼吸困難+浮腫が同時増悪 → 同時変化 → 同一疾患(心不全)
```
**同時に変化する症状は同じ疾患に属する** = Gestalt共通命運原則 = 聴覚時間相干性

### v3.2+v3.3 vs v4.x
- v3.2+v3.3で合併症の**大部分**解決（症状が十分異なる場合）
- v4.xは**最難ケース**（症状高度重疊）の最終解
- 時系列は嗅覚適応(背景消除)も同時に実現

## データ問題の再評価

### NMF補完は不可（証明済み）
- Netflix式補完は鑑別情報を系統的に破壊
- 形状向量は存在するが予測には使えない（r=0.24）

### 手動文献以外の選択肢
| 方法 | 可行性 | 瓶頸 |
|------|-------|------|
| 手動文献査証 | ✓ 現行 | 遅い、人力密集 |
| **EHR大数据** | ✓ 理論可行 | データアクセス権 |
| LLM抽出 | ✓ 粗粒度 | Bronze級精度 |
| NMF補完 | ❌ 証伪済 | 鑑別情報破壊 |

### EHR = 嗅覚の「進化学習」の対応物
- 嗅覚受体の選択 = 数億年の群体データ（進化）で最適化
- 物理法則 = 結合亲和力を免費計算（移植不可）
- EHR = 大量実患者データからCPTを直接測量(n/N実数計数)
- 「物理免費」は不可だが「進化学習」はEHRで可能

## 推論引擎の進化路線（感覚系統統合版）

```
v3.0: Noisy-OR + IDF（独立評分、補丁外挂）
v3.2: + D→D伝播（合併症因果鏈）← 万有因果図
v3.3: + explaining away（侧抑制）← 嗅覚
      + unexplained residual ← 免疫(先天免疫)
      - IDF削除
v4.x: + 時系列（適応 + 混合物分離）← 嗅覚+聴覚
      信息=変化(dX/dt)、時間相干性で合併症最終解決
```

**Why:** 嗅覚+聴覚+視覚+免疫の全感覚系統を系統的にVeSMedに対応付け。
**How to apply:** v3.3=嗅覚(侧抑制)+免疫(residual)。v4.x=嗅覚(適応)+聴覚(混合物分離)。データ=EHR(進化学習)。
