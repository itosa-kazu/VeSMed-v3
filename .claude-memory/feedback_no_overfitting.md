---
name: 辺追加の過拟合禁止
description: 辺監査で辺を追加する際、テスト案例のためだけの辺は追加禁止。臨床的合理性が独立して存在する辺のみ追加可。
type: feedback
---

辺監査で漏れ辺を修正する際、**テスト案例を通すためだけの辺追加は過拟合であり禁止**。

**判断基準:** 「この案例がなくても、この辺を追加するか？」
- Yes（教科書/文献で裏付け） → 追加OK
- No（この案例だけが理由） → 追加禁止

**合理的な辺追加の例:**
- D232 AKA → L54 低血糖: AKAはグリコーゲン枯渇で低血糖を起こす（教科書的事実）
- D246 EHEC → L14 血小板減少: EHEC-HUS合併で血小板減少は定義的

**過拟合の例（禁止）:**
- 感冒にL14(血小板減少)の辺を追加: たまたま案例のWBC panelにPLTが載っていたから
- CPT値を極端に設定(0.95等): 特定案例でTop-1にするため

**Why:** 辺監査は性能改善に非常に効果的（Top-1 +13~+18件）だが、やりすぎると431案例に過拟合し、未知の案例で性能低下するリスク。現在の案例数(477)でTop-3 93%は優秀だが、真の性能は未知案例でのみ測定可能。

**How to apply:** 辺追加時は常に臨床的根拠を自問。「この疾患はこの所見を本当に引き起こすか？頻度は何%程度か？」を考えてからCPT設定。

**逆方向の過拟合も禁止:** 臨床的に正しい辺を追加した後、特定案例が退化したからといって辺を撤回するのも過拟合。案例のデータが特殊（合併症等）なだけかもしれない。臨床事実に基づく辺は維持し、退化の原因は案例側を調査すること。
