## 疾患追加の標準方法論

### Step 1: 疾患選定
- 教科書/ガイドラインの鑑別リストから系統的に漏れを特定
- ER見逃し率データ等のエビデンスベースで優先度判断

### Step 2: Agent並行起動
- Agent A: PMC案例検索（疾患あたり3件以上）
- Agent B: 漏边監査（既存疾患の辺欠損チェック）
- Agent C-N: 複数疾患の案例を同時検索

### Step 3: 疾患モデル作成（三位一体 + R01/R02必須）
- step1（変数定義）+ step2（辺）+ step3（CPT）を同時追加
- **R01(年齢)とR02(性別)のfull_cptsを必ず追加** — 新疾患・疾患分割問わず必須
  - R01: 疫学文献から年齢7区分の発症率を設定
  - R02: 性差が明確なら必ず追加、不明でも50:50で追加推奨
  - per-R個別フォーマットで追加: `"R01": {"18_39": 0.01, ...}`
- 新変量が必要なら先に追加 + **IDF健全化（parent≧3、臨床的に合理的な辺のみ）**
- STATE_JAも忘れずに

### Step 4: 案例追加 + テスト
- **証拠マッピング検証**: vignette内の全所見がevidence dictに漏れなく入っているか確認。特にD-dimer/トロポニン/BNP等の新変量は入力漏れしやすい
- 伝統テスト（全所見一括 → Top-3確認）
- **ナビテスト（全案例で実施、省略禁止）**
- **回帰テスト：FATAL 0確認だけでなく、Top-1/Top-3の総数と百分率の変化を確認。悪化した案例があれば原因分析**

### Step 5: 漏边修正 + Top-3低下調査
- rank低い案例の原因分析 → 証拠にある変数への辺が欠損（NO EDGE）→ 辺追加
- CPT値の変更ではなく辺追加で解決（超参数なし）
- **Top-3百分率が低下した場合**: 非Top-3案例を全件リストアップ → 新疾患追加前と比較 → 新たに脱落した案例を特定 → 原因分析(新疾患がスコアを奪っているか？辺欠損か？) → 修正

### Step 5.5: Top-1低下の即時調査
- 新疾患追加後にTop-1が1件でも低下した場合、「軽微」と判断せず即時原因調査する
- 新疾患が既存案例のスコアを奪っている可能性がある → 辺追加/CPT調整で対応

- project_vesmed_status.md更新
- project_missing_diseases.md更新
- navigation_test_log.json更新
- sync_memory.sh実行 → git push

**Why:** 毎回同じ手順を踏むことでミスを防ぎ、品質を担保する。回帰テストはFATALだけでなくTop-1/Top-3の推移を見ないと、新疾患追加で既存案例が悪化していることに気づけない。
**How to apply:** 新疾患追加のたびにこのチェックリストに従う。
