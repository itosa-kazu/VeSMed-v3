## Netflix反論（致命的）

**Netflix とVeSMedの目標は正反対。**

- Netflix: 似た映画を推薦 → 類似性で補完 = 正しい
- VeSMed: 似た疾患を鑑別 → 類似性で補完 = **鑑別情報を破壊**

例: 肺炎と肺結核はほぼ同じCPTプロファイル（咳嗽、発熱、浸潤影）。
NMFは「肺炎≈肺結核」と学習し、欠損CPTを似た値で埋める。
しかし診断に必要なのは**差異**（盗汗は結核にあり肺炎にない）。
NMFは差異を系統的に補完できない — むしろ差異を消す方向に働く。

**Paper 1のconfident misdiagnosis問題と同構:**
- 辺欠失 = 類似疾患間の微細差異が未記録
- Netflix式補完 = 差異ではなく共通性を復元
- → 最も必要な情報を系統的に補完不能

## 反Netflix: 残差こそ鑑別情報

NMF予測値 = 「同類疾患ならこうなるはず」（無退屈な共通性）
残差 = 実際CPT − NMF予測 = **偏差 = 鑑別情報**

- 残差大(正) = この病は「意外にこの症状がある」→ 鑑別線索
- 残差大(負) = この病は「意外にこの症状がない」→ 鑑別線索
- 残差≈0 = 同類と同じ → 鑑別価値なし

**これはIDFの精密版:**
- IDF(全局): この症状は何疾患に出現するか
- 残差(局部): この症状はこの**類**の疾患に対して異常か

活用法:
- 既知CPT: 残差最大 = 鑑別金鉱 or データエラー → QCツール
- 未知CPT: NMF予測不安定な位置 = 鑑別情報が隠れている可能性 → 大清洗優先順位

## 嗅覚類比の最終評価

### 嗅覚の「データ」は二箇所
1. **受体蛋白（進化で最適化）** = 変量選択。数億年×全生物の生死 = 天文学的データ量
2. **物理法則（無料）** = CPT値。受体形状が決まれば、物理が全結合亲和力を同時決定

### VeSMedの対応
1. **変量選択** = 医学専門家。2000年の臨床知識の蓄積。進化ほどではないが、知的設計は盲目進化より効率的 — 一発で有用な変量を選べる
2. **CPT値** = 文献手動収集。物理法則に相当するものがない → **ここが根本的断裂点**

### 嗅覚から取れるもの/取れないもの

| 嗅覚の要素 | VeSMedでの実現 | データ依存? |
|-----------|--------------|-----------|
| 組合符号化 | ✓ 已有 | 不要 |
| 侧抑制/explaining away | v3.3で実装可 | 不要 |
| 適応（背景消除） | 時系列で実装可 | 不要 |
| 受体最適化 | 専門家選択で代替 | 少量 |
| **物理免費CPT** | **不可** | **大量必要** |

**結論: 嗅覚類比は架構（HOW）に有効、データ（WHAT）に無効。**
架構の洞察（侧抑制、適応、組合符号化）は実装可能で数億年のデータ不要。
CPT値の自動生成だけが不可能 — ここは人手+文献が不可避。

**Why:** 嗅覚形状互補仮説の実証実験+理論的限界の発見+残差活用の着想。
**How to apply:** 
- 低秩構造 → QCツール（異常CPT検出、大清洗優先順位付け）
- 架構洞察 → v3.2(D→D) + v3.3(explaining away) で実装
- CPT補完 → 不可。大清洗は人手必須
